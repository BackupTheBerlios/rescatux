File System Check (Forced Fix) - Rescapp - Documentation
  • Step 1

    Rescapp searchs for all the partitions based on information found in /proc/partitions file. If the partition contains an: /etc/issue file it will also obtain the distribution name.
  • Step 2

    Rescapp loops all the hard disks (based on information found in /proc/partitions. For each one of the found hard disks you are prompted:

    Rename hard disk if needed

    Detected: HARDDISK

    If the hard disk is correct please press OK button.

    If you know that the shown hard disk is called hda instead of detected sda you can write hda and press OK.
  • Step 3

    Rescatux redefines on the fly their detected hard disks.
  • Step 4

    Rescapp presents you found partitions on Step 1.

    You are prompted:

    Which partition do you want to Filesystem check with automatic fix?

    You are supposed to choose whatever partition you want to force filesystem check and press the OK button.

    Explanation of description is:
    • Distribution name: Your distribution root partition has been detected and you are seeing its name.
    • Not detected: The partition can be mounted but there is not a /etc/issue fie on it. Probably a non root partition.
    • Can't mount:: The partition cannot be mount. Maybe it is the partition you want to fix, another one that Rescatux cannot mount, or probably a SWAP partition.
  • Step 5

    While the fsck process is being run the following message is displayed:Running forced filesystem check and fix
  • Step 6

    You are informed about the fsck process.

    Message is self explanatory. You either get:

    Success! Filesystem check with automatic fix was OK! :)

    or

    Filesystem check with automatic fix went wrong! :(
  • Step 7

    You should click OK on the Running forced filesystem check and fix window.
  • Step 8

    A log file called fsck.log.txt can be found at Desktop logs folder.
